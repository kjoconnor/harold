Followed by a list of @github-username mentions, this means "[mentioned users] please review this pull request".

This is for initial review requests, not pinging people to come back like :haircut:.

Used by authors, but also reviewers when they want to summon other people.

{{ emoji["nail_care"] }} (:nail_care:)

Used by reviewers.

{{ emoji["haircut"] }} (:haircut:)

This means "I've responded or made appropriate changes and am ready for further review."

This doesn't accept usernames like :eyeglasses:, it just pings everyone who has reviewed so far. Use the GitHub review UI if you want to be more granular.

Used by authors.

{{ emoji["fish"] }} (:fish:)

This means "I think this pull request is ready for merge."

It's also OK to use this if the pull request is nearly ready to unblock the author when they make some final touchups.

Used by reviewers.

{{ emoji["running"] }} (:running:)

This means "I don't have time to review this at the moment, please find someone else."

Do this to be clear to authors that you aren't blocking their progress. But remember that code review is important and your input is helpful.

Used by reviewers.

{{ emoji["no_bell"] }} (:no_bell:)

When prefixing a comment, this means "This is just a comment and requires no action."
